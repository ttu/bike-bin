# Design: Migrate docs/specs to docs/design-docs

## Problem

Documentation in `docs/specs/` is fragmented and outdated. There are 9 ad-hoc design specs but no systematic per-feature documentation. The main docs in `docs/` describe the system at a high level but don't provide feature-level architecture detail that matches the actual codebase.

## Solution

Rename `docs/specs/` to `docs/design-docs/` and create 16 comprehensive design docs — one per feature slice plus cross-cutting concerns. Each doc is generated from the actual codebase (source code, migrations, routes, types), not copied from existing outdated docs.

## Document Template

Each design doc follows this structure:

```markdown
# Feature Name

## Overview

Brief description of what this feature does and why.

## Data Model

Tables, enums, relationships. Source of truth: migrations + TypeScript types.

## Architecture

Components, hooks, utils, providers — how the feature slice is organized.

## Screens & Navigation

Expo Router routes, screen purposes, navigation flows.

## Key Flows

Step-by-step user flows with data flow through the layers.

## RLS & Security

Relevant RLS policies and access control rules.

## i18n

Namespace, key structure.

## Current Status

What's implemented, what's planned, known gaps.
```

Sections are scaled to complexity — a feature with no database tables skips Data Model, a feature with no screens skips Screens & Navigation.

## Content Generation Approach

Each doc is generated by reading the actual source code:

1. **Feature slice** — `src/features/<name>/` directory structure, components, hooks, utils
2. **Types** — `src/shared/types/database.ts`, `models.ts`, `enums.ts`, `ids.ts`
3. **Migrations** — `supabase/migrations/*.sql` for tables, RLS policies, functions
4. **Routes** — `app/` directory for screens and navigation
5. **i18n** — translation JSON files for the feature namespace
6. **Existing specs** — merged where content is still accurate

## Side Effects

- **CLAUDE.md** — update documentation table: `docs/specs/README.md` → `docs/design-docs/README.md`
- **docs-sync.md** — update references from `docs/specs/` to `docs/design-docs/`

## Implementation Plan

This is a documentation-only change. No source code modifications.

### Phase 1: Setup (1 task)

- Rename `docs/specs/` → `docs/design-docs/`, update refs in CLAUDE.md and docs-sync.md

### Phase 2: Feature docs (14 parallel tasks)

- Generate 001-auth through 014-demo by reading each feature slice

### Phase 3: Cross-cutting docs (2 parallel tasks)

- Generate 015-design-system (consolidate 4 kinetic curator specs)
- Generate 016-rls-security (consolidate RLS spec + migration policies)

### Phase 4: Index and cleanup (1 task)

- Write README.md index, remove old individual spec files that were consolidated

## Risks

- **Content accuracy** — docs generated from code reading, not human domain knowledge. "Current Status" sections may miss context about what's intentionally incomplete vs just not built yet.
- **Maintenance burden** — 16 docs to keep in sync. Mitigated by the docs-sync command that validates drift.
